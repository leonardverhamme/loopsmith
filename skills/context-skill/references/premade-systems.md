# Premade Systems

Use this reference when grounding context work in proven public systems.

## Strong Public Patterns

### OpenAI Codex

- Persistent repo guidance belongs in `AGENTS.md`
- Codex works best with reliable commands and clear documentation

Sources:
- https://openai.com/index/introducing-codex/
- https://openai.com/business/guides-and-resources/how-openai-uses-codex/

### Anthropic Claude Code

- Codebase understanding is a first-class workflow
- Keep instructions concise and scoped
- Use subagents only when exploration truly branches

Sources:
- https://code.claude.com/docs/en/tutorials
- https://code.claude.com/docs/en/best-practices
- https://github.com/anthropics/skills

### MCAF

- Keep durable engineering context in the repo
- Separate AGENTS, architecture docs, and focused skills

Sources:
- https://mcaf.managed-code.com/
- https://mcaf.managed-code.com/skills

### Deep Wiki Pattern

- Repo-generated architecture maps, onboarding guides, and agent docs are a strong model for context upkeep

Sources:
- https://microsoft.github.io/skills/
- https://github.com/guaijie/deepwiki
