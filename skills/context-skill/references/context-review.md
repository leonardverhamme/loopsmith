# Context Review

Load this before stopping on any context-maintenance task.

## Checklist

- `AGENTS.md` stayed concise.
- Commands and paths are concrete.
- Durable context went into repo files, not just the chat.
- Existing docs were updated instead of duplicated.
- Any new repo map or architecture note is navigational, not bloated.
- Stale or contradictory guidance was removed or called out.

## Output Format

1. Context files updated
2. Main repo facts added or corrected
3. Remaining stale or missing context
